    Documentation Decidento API

    Bienvenue sur l'API DECIDENTO ! Vous pouvez utiliser l'API pour accéder aux différents points (endpoints) de Decidento, qui permettent de récupérer des informations sur divers Articles, Entreprises, Signaux, et Contacts présents dans notre base de données.

    Pour utiliser l'API Decidento, consultez la section d'authentification et rendez-vous sur API Decidento pour tester.

    Path Table

    La structure de la réponse Json va être affiché dans cette section !

    Method Path Description
    GET /articles Obtenir les derniers articles
    GET /articles/search Rechercher une liste d'articles
    GET /articles/search Trouver un article par ID
    GET /articles/collection Trouver une collection d'articles par IDS
    GET /companies Rechercher des entreprises avec des paramètres
    GET /companies/{companyId} Trouver une entreprise par ID

    GET /signals Obtenir les derniers signaux
    GET /signals/{signalId} Trouver un signal par ID
    GET /signals/collection Trouver une collection de signaux par IDS

    Authentification

    Utilisation

    Pour accéder aux services de l'API, vous devez d'abord obtenir une clé d'API. Enregistrez-vous ici pour créer un compte. Une fois que vous êtes inscrit, vous pourrez récupérer vos clés d'authentification depuis votre dashboard personnel.

    Comment s'authentifier ?

    Pour chaque requête à l'API, vous devez inclure les en-têtes d'authentification suivants :

    • X-AUTH-ID : C'est l'identifiant unique lié à votre compte, obtenu après inscription.
    • X-AUTH-TOKEN : Ce jeton est associé à votre compte et garantit l'accès sécurisé aux services de l'API.

    Articles

    L'API "Articles" permet d'interagir avec des articles via différentes requêtes HTTP. Vous pouvez récupérer les derniers articles, effectuer des recherches, ou consulter des collections d'articles à l'aide d'IDs spécifiques. Voici les principales routes :

    • GET /articles : Récupère les derniers articles disponibles.
    • GET /articles/search : Permet de rechercher une liste d'articles en fonction de critères spécifiques.
    • GET /articles/{articleId} : Trouve un article précis à partir de son identifiant unique (ID).
    • GET /articles/collection : Récupère une collection d'articles en fournissant une liste d'IDs.

    This endpoint retrieves a list of articles according to the query parameters shown below.

    HTTP Request

    GET https://api.decidento.com/articles

    Query Parameters

    Champ Type Description
    term chaîne de caractères Terme de recherche (mots-clés, siren, nom social, ...)
    siren entier Siren des entreprises principales liées à l'article / aux signaux (format : ^([0-9]{9}))
    date_begin chaîne de caractères Date de début de publication (ex: 2019-03-03)
    date_end chaîne de caractères Date de fin de publication (ex: 2019-03-03)
    limit entier Nombre maximal d'enregistrements à retourner. Si la valeur dépasse le maximum (200), le maximum sera utilisé.
    Valeur par défaut : 100
    offset entier Numéro de départ pour les articles retournés. Utile pour paginer les résultats.
    Valeur par défaut : 1
    order chaîne de caractères Champ de tri.
    Valeur par défaut : publishedAt
    sort chaîne de caractères Sens du tri.
    Valeurs disponibles : desc, asc.
    Valeur par défaut : desc
    meta booléen Afficher les informations de pagination.
    Valeur par défaut : false
    _format chaîne de caractères Format de sortie.
    Valeurs disponibles : json, xml.
    Valeur par défaut : json
    X-API-VERSION chaîne de caractères Version de l'API (X-API-VERSION)

    Companies

    L'API "Companies" permet de rechercher et de récupérer des informations détaillées sur des sociétés à partir de plusieurs types d'identifiants et de paramètres. Voici les principales routes disponibles :

    • GET /companies : Recherche de sociétés en utilisant divers paramètres de recherche.
    • GET /companies/{companyId} : Récupère les détails d'une société spécifique à partir de son identifiant unique (ID).
    • GET /companies/collection : Récupère une collection de sociétés en fournissant une liste d'IDs.
    • GET /companies/v2 : Nouvelle version pour la recherche de sociétés avec des paramètres avancés.
    • GET /companies/{companyCccid}/v2 : Trouve une société en utilisant son identifiant unique CCCID.
    • GET /companies/collection/v2 : Récupère une collection de sociétés en fournissant une liste de numéros SIREN.
